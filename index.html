MongoDB常用指令

Posted on 2017-05-15

  • 查询 db.col.find().pretty()
    OR: db.col.find(
    {$or: [{key1: value1}, {key2:value2}]}).pretty()
    AND: db.col.find({key1:value1, key2:value2}).pretty()

  • 插入 db.col.insert() / db.col.save(document)

  • 更新 db.col.update({查询条件},{跟新的内容}，{upsert:ture}{mulit,ture})

    mulit：flase只更新找到的第一条记录,更新查询到的所有

    upsert:如果不存在update的记录, 是否插入
  • 删除 db.col.remove({query},2) 2代表删除的条数
  • 删除数据库 db.dropDatabase()
  • 创建/连接数据库 use db
  • 查看所有数据库 show dbs
  • 删除集合 db.collection.drop()
    使用用户名和密码连接登陆到指定数据库：mongodb://admin:123456@localhost/test

  • 条件操作符

    $lt <

    $lte <=
    $gt >
    $gte >=

    $ne !=

  • type 字段类型查询 db.col.find({“title” : {$type : 2}}) 其中2为string

  • skip()略过
    limit()指定查询的条数
    sort()排序 db.COLLECTION_NAME.find().sort({KEY:1}) key为1 or -1 指定排序的方式

  • 索引:MongoDB使用ensureIndex()方法来创建索引
    db.COLLECTION_NAME.ensureIndex({KEY:1}),Key 值为你要创建的索引字段，1为指定按升序创建索引，降序索引指定为-1.
    在后台创建索引实例：
    db.values.ensureIndex({open: 1, close: 1}, {background: true})
    通过在创建索引时加background:true 的选项，让创建工作在后台执行

Github + Hexo

Posted on 2017-05-15

Github 和 个人域名绑定

  • 第一步,在阿里云购买自己的域名,在域名列表中选中需要绑定的域名解析,如下图:

  • 第二步,填写你的IP解析，此处的IP是你github的,如我的是paul15342.github.io


    然后添加一条解析记录类型是CNAME,记录值是paul15342.github.io


  • 第三步,登录GitHub，进入之前创建的仓库，点击settings，设置Custom domain，输入你的域名,保存


  • 第四步, 进入本地博客文件夹 ，进入hexo/source目录下，创建一个记事本文件，输入你的域名，只填写域名即可,保存，命名为CNAME ，注意保存成所有文件（不带任何后缀）



    然后在git bash中一次完成下面三个指令

    hexo clean

    hexo g

    hexo d

打开浏览器输入你的域名，即可进入到你的博客页面

My first Blog

Posted on 2017-05-14

#My first Blog

搭建github + Hexo 个人博客

一:安装 (Hexo中打开Git Bash Here)

输入一下密令:
npm install -g hexo

二:初始化

在电脑的某个地方新建一个名为hexo的文件夹（名字可以随便取），比如我的是F:hexo，由于这个文件夹将来就作为你存放代码的地方，所以最好不要随便放。

hexo init

hexo g

hexo s #启动服务

三:上传到github

首先，配置好github, ssh key肯定要配置好。

其次，配置_config.yml中有关deploy的部分：

repo是在github中新建的资源库的链接

正确写法：

deploy:
type: git

repository: git@github.com:liuxianan/liuxianan.github.io.git

branch: master

然后执行hexo deploy
此时一般会报如下错误：
Deployer not found: github 或者 Deployer not found: git
原因是还需要安装一个插件：

npm install hexo-deployer-git –save

_config.yml
这里面都是一些全局配置，每个参数的意思都比较简单明了，所以就不作详细介绍了。

需要特别注意的地方是，冒号后面必须有一个空格，否则可能会出问题。

###四:github中建立资源库
在github中建立资源库
https://github.com/new

这个新仓库就将是存放你即将拥有的博客的地方

注意，你的仓库名不能随便取，这样会导致github混乱，
取名的格式应该为“用户名.github.io”

###删除文章
本地删除后， hexo g ,hexo d

四:Hexo常用指令：

  • hexo new “postName” #新建文章
  • hexo new page “pageName” #新建页面
  • hexo generate #生成静态页面至public目录
  • hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
  • hexo deploy #部署到GitHub
  • hexo help # 查看帮助
  • hexo version #查看Hexo的版本

缩写

  • hexo n == hexo new
  • hexo g == hexo generate
  • hexo s == hexo server
  • hexo d == hexo deploy

组合命令：

  • hexo s -g #组合并生成本地预览
  • hexo d -g #生成并上传
